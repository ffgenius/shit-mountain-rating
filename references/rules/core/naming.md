# 命名规范评分规则

分析命名规范。

## 检测项

- 拼音命名（使用拼音而非英文命名）
- 无意义变量名（a, b, x, tmp, data, info, obj, ret, val 等单字/泛化名）
- 过度缩写（<4 字符且无法从上下文推断含义）
- 命名风格不统一（camelCase / snake_case / PascalCase 混用在同一作用域）
- 同一概念多种命名（如 userId / userID / user_id 表示同一概念）
- 文件名与内容严重不匹配（utils.ts 里有数据库操作）
- 拼写错误（常见英文单词拼错）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| 拼音命名 | 1-2 处 | 3 处及以上 |
| 无意义变量 | 3-5 处 | >5 处 |
| 过度缩写 | 3-5 处 | >5 处 |
| 风格混用 | 同一文件 2 种 | 同一文件 3 种以上 |
| 同义异名 | 同一概念 2 种 | 同概念 3 种以上 |
| 文件名误导 | 1 个文件 | 3 个以上 |
| 拼写错误 | 1-2 处 | 3 处以上 |

## 评分公式

```
分数 = 100 - (拼音命名严重 × 8) - (无意义变量严重 × 6) - (过度缩写严重 × 4) - (风格混用严重 × 5) - (同义异名严重 × 5) - (文件名误导严重 × 4) - (拼写错误严重 × 3) - (警告 × 2)
```

最低 0 分。

## 检测方式

- 变量名与常用英文单词表对比检测拼音
- 变量名长度和上下文语义分析
- 同一文件/项目内命名风格统计
- 文件名与导出符号的语义匹配
- 常见拼写错误词库对比
