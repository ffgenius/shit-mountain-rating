# 性能评分规则

分析性能问题。

## 检测项

- N+1 查询（循环内数据库查询）
- 阻塞 I/O（同步文件读写、同步网络请求在主线程）
- 内存泄漏（未释放的引用、未清理的定时器/事件监听、闭包持有大对象）
- 重复计算（循环内不变的计算、每次渲染都重新计算相同值）
- 大对象复制（不必要的深拷贝、大数组展开）
- 循环内创建对象（频繁触发 GC）
- 缺少缓存（重复请求相同数据、重复计算相同结果）
- 重复 JSON 序列化/反序列化（同一个对象反复转换）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| N+1 查询 | — | 任意出现 |
| 阻塞 I/O | 非关键路径 | 关键路径出现 |
| 内存泄漏 | 潜在风险 | 已确认泄漏 |
| 重复计算 | 循环内 1 处 | 循环内 >2 处 |
| 大对象复制 | 1-2 处 | 3 处以上 |
| 循环内创建对象 | 1-2 处 | 3 处以上 |
| 缺少缓存 | 低频数据 | 高频数据无缓存 |
| 重复序列化 | 1-2 处 | 3 处以上 |

## 评分公式

```
分数 = 100 - (N+1 × 15) - (阻塞IO严重 × 10) - (内存泄漏 × 20) - (重复计算严重 × 5) - (大对象复制严重 × 5) - (循环创建对象严重 × 5) - (缺缓存严重 × 8) - (重复序列化严重 × 5) - (警告 × 2)
```

最低 0 分。

## 检测方式

- 检测循环内数据库查询/HTTP 请求
- 识别同步 I/O 调用点
- 事件监听器/定时器注册与清理配对检查
- 循环内 new/对象字面量/malloc 检测
- 相同请求/计算结果重复出现检测
