# 可读性评分规则

分析代码可读性。

## 检测项

- 超长函数（>50 行警告，>100 行严重）
- 超长文件（>300 行警告，>500 行严重）
- 超长表达式（单行 >120 字符或链式调用 >5 层）
- 深层嵌套（>3 层警告，>5 层严重）
- 复杂条件表达式（单个 if 条件 >3 个逻辑运算符）
- 变量声明远离使用位置（>20 行距离警告，>30 行严重）
- 代码块缺少空行分隔（连续 >30 行无空行）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| 函数行数 | >50 | >100 |
| 文件行数 | >300 | >500 |
| 嵌套深度 | >3 | >5 |
| 表达式长度 | >120 字符 | >200 字符或链式 >5 层 |
| 条件复杂度 | >2 个逻辑运算符 | >3 个 |
| 变量距离 | >20 行 | >30 行 |
| 连续无空行 | >30 行 | >50 行 |

## 评分公式

```
分数 = 100 - (超长函数严重 × 8) - (超长文件严重 × 5) - (深层嵌套严重 × 6) - (超长表达式严重 × 3) - (复杂条件严重 × 4) - (变量距离严重 × 3) - (缺空行严重 × 3) - (警告 × 2)
```

最低 0 分。

## 检测方式

- 逐函数/文件统计行数
- AST 分析嵌套深度和表达式复杂度
- 检测连续无空行的代码段
- 变量作用域内首次使用与声明位置的距离
