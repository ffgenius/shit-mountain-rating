# 💩 屎山分析器 (Shit Mountain Rating)

> 让一个被折磨了十五年的老程序员帮你骂代码。

识别技术债、量化评分、输出毒舌重构报告。覆盖 15 个评分维度，支持 9 种编程语言和 9 个主流框架的专项检查 (陆续补充中...)。

## 安装

```bash
pnpx skills add ffgenius/shit-mountain-rating
```

安装完成后在 Claude Code 中即可使用。

## 快速开始

```
/shit-mountain-rating                    # 分析当前项目
/shit-mountain-rating --path ./src       # 分析指定路径
/shit-mountain-rating --quick            # 快速扫描（仅核心规则，跳过语言/框架专项）
```

分析完成后会在终端输出完整报告，包含综合评分、各维度得分、Top 10 问题和重构优先级。

## 使用场景

### 场景一：接手遗留项目，想知道有多烂

```bash
/shit-mountain-rating --path ./shit-mountain-rating-billing-system
```

输出示例：

```
# 📊 遗留系统分析报告

## 项目概况
- 主要语言: Java
- 主要框架: Spring Boot 2.1 (EOL!)
- 文件总数: 847
- 代码总行数: 126,340

## 整体健康度: 34/100 ☠ 考古遗址
```

然后你会得到一份毒舌总结，比如：
> "265 行的 Service 类里塞了 15 个 God Function，每个都在说'我还能再扛一个需求'。"

### 场景二：Code Review 前的快速体检

```bash
/shit-mountain-rating --quick --path ./feature/new-payment-module
```

快速扫描跳过语言和框架专项规则，只跑核心规则（架构、复杂度、安全、错误处理等 13 项），适合 PR 合并前的快速检查。

### 场景三：重构规划

```bash
/shit-mountain-rating --path ./src
```

完整报告包含按优先级排列的重构建议：

| 优先级 | 说明 | 示例问题 |
|--------|------|----------|
| P0 | 立即处理 — 安全漏洞、数据丢失风险 | SQL 注入、未处理异常导致崩溃 |
| P1 | 高优先级 — 严重影响可维护性 | God Object、循环依赖 |
| P2 | 建议优化 — 影响开发效率 | 重复代码、命名不规范 |
| P3 | 长期优化 — 低风险改进 | 注释过时、测试覆盖率 |

### 场景四：技术债盘点

```bash
/shit-mountain-rating --path .
```

报告会统计并分类所有技术债标记：

- `FIXME` / `TODO` / `HACK` / `XXX` / `@deprecated` 的数量和分布
- 死代码和注释掉的代码块
- 废弃接口和未使用的依赖

### 场景五：多项目横向对比

```bash
/shit-mountain-rating --path ./project-a
/shit-mountain-rating --path ./project-b
/shit-mountain-rating --path ./project-c
```

用统一的 15 维度评分体系横向对比多个项目，识别最需要治理的代码库。

## 评分体系

### 15 个评分维度

| 维度 | 权重 | 检查内容 |
|------|------|----------|
| 🔒 安全性 | 14% | SQL 注入、XSS、硬编码密钥、命令注入 |
| ⚠️ 错误处理 | 11% | Empty Catch、未处理 Promise、静默错误 |
| 🏗️ 架构质量 | 9% | 分层、模块职责、God Object |
| 🔗 耦合度 | 8% | 循环依赖、跨层依赖、Import 密度 |
| 🔧 可维护性 | 8% | 模块独立性、可扩展性、修改影响范围 |
| 🌀 代码复杂度 | 6% | 圈复杂度、嵌套深度、文件大小 |
| ⚡ 性能 | 6% | N+1 查询、阻塞 I/O、内存泄漏 |
| 🧪 测试质量 | 6% | 测试覆盖率、Mock 质量、可测试性 |
| 📖 可读性 | 6% | 超长函数、深层嵌套、超长文件 |
| 📋 重复度 | 5% | 重复逻辑、重复 SQL、相似函数 |
| 💸 技术债 | 5% | 死代码、废弃接口、注释掉的代码块 |
| 📝 语言规范性 | 5% | 语言层面反模式严重程度 |
| 🖼️ 框架规范性 | 5% | 框架层面反模式严重程度 |
| 🏷️ 命名规范 | 4% | 拼音、无意义变量、缩写、命名不统一 |
| 💬 注释标记 | 2% | FIXME/TODO/HACK/XXX/@deprecated 统计 |

### 综合评级

| 分数 | 评级 | 含义 |
|------|------|------|
| 95+ | 🏆 工程杰作 | 教科书级别的代码，可以拿去面试炫耀 |
| 90+ | ✨ 卓越 | 质量很高，继续保持 |
| 80+ | 😊 健康 | 整体不错，有小毛病 |
| 70+ | 🙂 可维护 | 能正常工作，但技术债在悄悄积累 |
| 60+ | 😐 技术债堆积 | 已经到了该还债的时候了 |
| 50+ | 😵 遗留系统 | 每次加需求都要烧香拜佛 |
| 40+ | 💀 需要重构 | 重构成本已经超过重写成本的一半 |
| 20+ | ☠️ 遗留地狱 | 能跑起来就是个奇迹 |
| 0+ | 🔥 考古遗址 | 这代码应该进博物馆，不该进生产环境 |

## 支持的语言和框架

### 编程语言（9 种）

TypeScript · JavaScript · Rust · Go · Java · Python · C# · C++ · CSS

### 框架（9 个）

React · Vue · Angular · NestJS · Spring · Node.js · Express · Django · Flask

分析器会自动检测项目使用的语言和框架，加载对应的专项规则。规则存放在 `rules/language/` 和 `rules/framework/` 目录下。

## 报告结构

每次分析输出六部分：

1. **项目概况** — 语言、框架、文件数、代码行数
2. **各维度评分** — 15 个维度的分数和星级
3. **Top 10 问题** — 按严重程度排列的最烂代码
4. **风险分析** — 高风险模块、核心路径问题、安全隐患
5. **重构优先级** — P0 到 P3 四级优先级的可操作建议
6. **毒舌总结** — 一句扎心但不侮辱人的吐槽

## 自定义规则

所有规则以 Markdown 文件存放，可直接编辑：

- **调整权重** — 编辑 `metrics/overall.md` 中的权重表
- **添加新语言** — 在 `rules/language/` 下创建新的 `.md` 文件
- **添加新框架** — 在 `rules/framework/` 下创建新的 `.md` 文件
- **修改评分标准** — 编辑 `metrics/` 下对应维度的文件
- **调整毒舌风格** — 编辑 `persona.md` 和 `report/sarcasm.md`

### 添加自定义规则示例

假设要为 Kotlin 添加语言规则，创建 `rules/language/kotlin.md`：

```markdown
# Kotlin 语言专项规则

## 检查项

1. **数据类滥用** — data class 超过 20 个字段扣分
2. **Scope 函数嵌套** — let/run/apply/also 链式调用超过 3 层扣分
3. **Null 安全滥用** — 过度使用 `!!` 强制解包扣分
4. **协程异常处理** — launch 内部未捕获异常扣分

## 评分标准

每个检查项按严重程度扣 1-10 分，满分 100 分扣完为止。
```

## 常见问题

### Q: 快速扫描和完整扫描的区别？

快速扫描 (`--quick`) 只执行 13 条核心规则（架构、复杂度、耦合、安全等语言无关项），跳过语言和框架专项检查。速度快但覆盖不全，适合日常快速检查。

完整扫描执行核心规则 + 语言规则 + 框架规则，共三轮检查，适合里程碑节点的全面审计。

### Q: 分数太低怎么办？

别慌。先看 P0 问题（安全漏洞、数据风险），修完再看 P1（God Object、循环依赖）。不要试图一次性修完所有问题 —— 重构是一场马拉松。

### Q: 毒舌总结能关掉吗？

不能。这就是这个工具的特色。但记住：骂的是代码，不是你。

### Q: 支持自定义语言和框架吗？

支持。在 `rules/language/` 或 `rules/framework/` 下添加对应的 `.md` 规则文件即可，分析器会自动加载。
