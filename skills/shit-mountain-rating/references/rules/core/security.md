# 安全评分规则

分析安全问题。

## 检测项

- SQL 注入（拼接 SQL 语句）
- 命令注入（用户输入拼入系统命令）
- 硬编码密钥（密码/Token/API Key/AES Key 写死）
- XSS（未转义的用户内容输出到 HTML）
- CSRF（状态变更操作缺少防护令牌）
- Eval / 动态执行（eval、new Function、exec、setTimeout 字符串版）
- 路径遍历（用户输入拼入文件路径 `../../../etc/passwd`）
- 文件上传未校验（类型、大小、内容未检查）
- 敏感数据明文存储/传输（密码明文、HTTP 传敏感信息、日志中打印凭据）
- CORS 配置过于宽松（`*` 且 Allow-Credentials）
- 依赖库已知漏洞（有 CVE 的过期版本）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| SQL 注入 | — | 每处即严重 |
| 命令注入 | — | 每处即严重 |
| 硬编码密钥 | — | 每处即严重 |
| Eval | — | 每处即严重 |
| 路径遍历 | — | 每处即严重 |
| 文件上传未校验 | — | 每处即严重 |
| XSS | 1-2 处 | 3 处以上 |
| CSRF | 缺少部分防护 | 无任何防护 |
| 敏感信息泄露 | 1 处 | 2 处以上 |
| CORS 宽松 | — | 生产环境出现 |
| 依赖漏洞 | 低危 CVE | 高危/Critical CVE |

## 评分公式

```
分数 = 100 - (注入漏洞 × 25) - (命令注入 × 25) - (硬编码密钥 × 20) - (Eval × 20) - (路径遍历 × 20) - (文件上传 × 15) - (XSS严重 × 10) - (CSRF严重 × 10) - (敏感信息严重 × 15) - (CORS × 10) - (依赖漏洞严重 × 15)
```

最低 0 分。注入类每处扣 25 分——可直接拿 shell 的问题。

## 检测方式

- SQL 语句拼接模式检测
- 命令执行中的变量拼接检测
- 密钥/Token 正则扫描
- 文件路径拼接中的 `..` 和变量检测
- CORS/CSP 头配置分析
- 依赖版本与 CVE 数据库对比
- 日志输出中敏感字段检测（password、token、secret、key）
