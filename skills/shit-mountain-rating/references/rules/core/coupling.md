# 耦合度评分规则

分析模块耦合度。

## 检测项

- Import 数量（单文件依赖数）
- Export 数量（单文件被依赖数）
- 循环依赖（Circular Dependency，模块间互相引用）
- 跨模块依赖（Cross Module Dependency）
- 依赖密度（被依赖数 / 依赖数的失衡程度）
- 中转依赖（A→B→C，A 不需要 B 的任何符号）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| 单文件 Import 数 | >10 | >15 |
| 循环依赖 | — | 任意数量即严重 |
| 跨层依赖 | 1 处 | >2 处 |
| 依赖密度失衡 | 被依赖/依赖 <0.3 | <0.1 |
| 中转依赖 | 1-2 处 | >2 处 |

## 评分公式

```
分数 = 100 - (循环依赖 × 20) - (高Import严重 × 8) - (跨层依赖严重 × 10) - (密度失衡严重 × 5) - (中转依赖严重 × 5)
```

最低 0 分。循环依赖每个扣 20 分——这是架构级硬伤，没有商量余地。

## 检测方式

- 构建依赖图
- 拓扑排序检测循环
- 统计每个文件的 import/export 数量
- 检查最短路径中是否存在无用跳板
