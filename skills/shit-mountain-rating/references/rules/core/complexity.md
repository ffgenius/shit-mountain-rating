# 复杂度评分规则

分析代码复杂度。从理解和测试难度视角评分，与架构维度的"职责分配"视角互补。

## 检测项

- 文件行数
- 函数行数
- 类行数
- 圈复杂度（Cyclomatic Complexity）
- 认知复杂度（Cognitive Complexity）
- 嵌套深度（Nesting Depth）
- 分支数（Branch Count）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| 文件行数 | >300 | >500 |
| 函数行数 | >50 | >100 |
| 类行数 | >300 | >500 |
| 圈复杂度 | >10 | >20 |
| 认知复杂度 | >15 | >30 |
| 嵌套深度 | >3 | >5 |
| 分支数 | >5 | >10 |

## 评分公式

```
分数 = 100 - (严重超限函数 × 10) - (严重超限文件 × 8) - (警告超限文件 × 3) - (高嵌套函数 × 5) - (高分支函数 × 5)
```

- 严重超限函数：行数 >100 或 圈复杂度 >20
- 严重超限文件：行数 >500
- 警告超限文件：行数 >300
- 高嵌套函数：嵌套深度 >5
- 高分支函数：分支数 >10

最低 0 分。

## 检测方式

- 逐文件统计行数
- 逐函数计算圈复杂度和认知复杂度
- AST 分析嵌套深度和分支数
- 结果排序取 Top N 命中项，只对最严重的一个档次计分（同一条函数只取最高扣分项）
