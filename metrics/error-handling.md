# 错误处理评分

错误处理质量评分。

## 评分维度

- Empty Catch 数量（吞掉异常无处理）
- 未 catch 的 Promise 数量
- throw 非 Error 对象数量（throw String）
- 静默错误数量（日志都没有的错误处理）
- 缺少重试机制的网络/IO 操作

## 评分公式

```
分数 = 100 - (Empty Catch × 20) - (未处理Promise × 20) - (throw String × 10) - (静默错误 × 15) - (缺重试 × 5)
```

最低 0 分。Empty Catch 和未处理 Promise 各扣 20 分——都是生产事故的直接来源。

## 星级映射

| 分数 | 星级 | 评级 |
|------|------|------|
| 90+ | ★★★★★ | 错误处理完善 |
| 75+ | ★★★★☆ | 错误处理良好 |
| 60+ | ★★★☆☆ | 有隐患 |
| 40+ | ★★☆☆☆ | 错误处理薄弱 |
| 0+  | ★☆☆☆☆ | 定时炸弹 |
