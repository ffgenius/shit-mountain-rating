# 工作流程 — 分析流程

## 流程图

```
识别项目
    │
    ▼
识别语言
    │
    ▼
识别框架
    │
    ▼
执行核心规则 ──────────────┐
    │                         │
    ▼                         ▼
执行语言规则              扫描文件结构
    │                         │
    ▼                         ▼
执行框架规则              识别依赖关系
    │
    ▼
计算评分 (评分计算)
    │
    ▼
输出报告 (报告模板)
```

## 各阶段说明

### 阶段一：项目识别
- 扫描目录结构
- 识别项目类型（Monorepo / 单体 / 微服务）
- 统计文件数量、代码行数

### 阶段二：语言识别
- 检测主要编程语言
- 识别多语言混合项目
- 确定主导语言

### 阶段三：框架识别
- 检测使用的框架（通过依赖文件）
- package.json / Cargo.toml / go.mod / pom.xml 等
- 识别框架版本

### 阶段四：规则执行
- 核心规则 → 语言无关的通用分析
- 语言规则 → 语言特定问题
- 框架规则 → 框架最佳实践检查

### 阶段五：评分计算
- 各维度独立评分
- 加权计算综合评分
- 生成评级

### 阶段六：报告输出
- 按模板组织输出
- 应用人格设定风格
- 输出重构优先级
