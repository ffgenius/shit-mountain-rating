# 死代码评分规则

分析死代码。

## 检测项

- 未引用文件（从入口文件不可达的文件）
- 未使用函数（定义但从未被调用的函数/方法）
- 未使用变量（声明/赋值但从未读取）
- 废弃接口仍在使用（@deprecated 接口仍在被调用）
- Legacy API（旧版 API 仍在使用且有新替代）
- 被注释掉的代码块（>10 行连续注释掉的代码）
- 不可达代码（return/throw/break 之后的代码）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| 未引用文件 | 1-3 个 | >3 个 |
| 未使用函数 | 3-5 个 | >5 个 |
| 未使用变量 | 5-10 个 | >10 个 |
| 废弃接口仍用 | 1-2 个 | >2 个 |
| Legacy API | 1-2 个 | >2 个 |
| 注释掉的代码块 | 1-3 处 | >3 处 |
| 不可达代码 | 1-2 处 | >2 处 |

## 评分公式

```
分数 = 100 - (未引用文件严重 × 10) - (未使用函数严重 × 6) - (废弃接口严重 × 8) - (Legacy API严重 × 8) - (注释代码块严重 × 5) - (不可达代码严重 × 5) - (警告 × 2)
```

最低 0 分。

## 检测方式

- 从入口文件构建引用图，标记不可达文件
- Tree-shaking 分析标记未使用导出
- AST 分析未读取的变量声明
- @deprecated 标记和调用点扫描
- 连续注释行检测
- 控制流分析检测不可达代码
