# 测试质量评分规则

分析测试质量。

## 检测项

- 测试覆盖率（行覆盖率 + 分支覆盖率）
- Mock 质量（外部依赖是否合理隔离）
- 全局状态（单例、全局变量、模块级可变状态阻碍测试）
- 难测试代码（需要大量 mock 才能测试的逻辑）
- 测试与实现强耦合（改一行实现崩一片测试——脆弱的测试）
- 假测试（无断言或 `expect(true).toBe(true)` 类占位测试）
- 测试金字塔失衡（只有单元测试无集成测试，或反过来）

## 阈值定义

| 指标 | 警告 | 严重 |
|------|------|------|
| 行覆盖率 | <60% | <30% |
| 分支覆盖率 | <50% | <20% |
| 全局状态 | 1-2 处 | 3 处及以上 |
| 难测试代码 | 1-2 个模块 | 3 个以上 |
| 测试与实现强耦合 | — | 任意出现 |
| 假测试 | — | 任意出现 |
| 测试金字塔 | 偏科 | 完全失衡 |

## 评分公式

```
分数 = 100 - (零测试模块 × 20) - (全局状态严重 × 10) - (难测试模块严重 × 15) - (耦合测试 × 10) - (假测试 × 15) - (覆盖率严重 × 10) - (测试金字塔 × 8)
```

最低 0 分。

## 检测方式

- 覆盖率报告分析（行 + 分支）
- 扫描全局变量、单例模式、模块级可变状态
- 测试文件与被测文件 change coupling 分析
- 扫描空断言和仅返回 true 的测试用例
