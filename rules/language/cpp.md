# C++ 语言规范

C++ 语言专项检查。

## 检测项

- 裸指针 `new` / `delete`（应使用 `std::unique_ptr` / `std::shared_ptr`）
- 手动内存管理（malloc/free 混用 new/delete，或者不配对）
- `NULL` 或 `0` 代替 `nullptr`（函数重载时 nullptr 的类型更精确）
- C 风格类型转换（`(int)x` 应使用 `static_cast<int>(x)`）
- 头文件循环依赖（A.h include B.h，B.h include A.h）
- 头文件中 `using namespace std`（污染所有 include 这个头文件的翻译单元）
- 未初始化的变量（栈上的基本类型和指针，值是随机的）
- 缺少虚析构函数（基类无 virtual ~Foo()，delete 基类指针时子类析构不执行）
- 自己管理的原始数组（`int arr[100]` 而非 `std::array` 或 `std::vector`）
- 三/五法则违反（自定义了析构/拷贝/移动之一但没有定义其他几个）
- 异常不安全代码（在构造中分配资源后抛异常，前面的资源无人释放）
- 不合理的 `const` 缺失（该加 const 的成员函数和参数都不加）
- `#define` 宏函数（应用 constexpr 或 inline 函数代替）
- 成员变量访问控制缺失（所有成员 public，类沦为一堆响数据的容器）
- 字符串用 `char*` 而非 `std::string`（自己管理 buffer 长度和内存）
- 多线程中访问共享变量无同步（data race，UB）
- `std::move` 滥用（move 了之后继续用原对象）
- 构造函数中调用虚函数（子类还没构造完成，虚表指向父类）
- `void*` 指针（丢掉类型信息，使用时再强制转回）
- 全局对象（不同翻译单元的初始化顺序不确定）
- `static_cast` 做向下转型不用 `dynamic_cast`（多继承或多态场景下指针偏移错误）

## 评分标准

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| 多线程无同步 | 严重 | 未定义行为，数据竞争 |
| 构造函数调虚函数 | 严重 | 运行时调到父类版本，子类的覆写永远不被调用 |
| 三/五法则违反 | 严重 | 双重释放、浅拷贝导致的多次释放、悬空指针 |
| 异常不安全 | 严重 | 构造中途抛异常，已分配资源泄漏 |
| 缺少虚析构 | 严重 | delete 基类指针时子类部分析构被跳过 |
| new/delete 裸指针 | 高 | 所有权不清晰，异常路径下忘记 delete |
| 未初始化变量 | 高 | 栈上的值不确定，但直接使用是 UB |
| 头文件 using namespace | 高 | 命名空间污染，导致符号冲突 |
| 头文件循环依赖 | 高 | 编译错误或 ODR 违反 |
| C 风格转换 | 高 | 编译器不检查转换合理性，reinterpret 滥杀伤 |
| 全局对象 | 高 | 初始化顺序不确定引发偶发崩溃 |
| 宏函数 | 中 | 参数求值多次、无作用域、无类型检查 |
| void* | 中 | 类型信息丢弃 |
| 原始数组 | 中 | 越界无检测、长度不传递 |
| 手动内存管理 malloc/free | 中 | 与 new/delete 行为不同 |
| NULL/0 代替 nullptr | 中 | 重载解析可能选择非预期版本 |
| const 缺失 | 中 | 代码意图不明确，误改的风险 |
| move 后再用 | 中 | 使用被移动的对象是合法但值不确定 |
| 成员全 public | 低 | 封装被破坏 |
| 不用 string | 低 | 在特定场景可接受，但绝大多数情况下 std::string 更安全 |
| static_cast 向下转型 | 低 | 特定场景才出错 |
